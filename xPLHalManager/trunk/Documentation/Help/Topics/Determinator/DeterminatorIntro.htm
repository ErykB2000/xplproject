Introduction to Determinators

A typical xPL environment will consist of a number of devices. Some of these devices will report information, some will act on xPL messages, and some will do both. For example, a CM12 X10 computer interface reports to the xPL network when it receives an X10 signal, and can also transmit X10 signals in response to a command from the xPL network.

A Determinator is simply a set of conditions, and a corresponding set of actions. For example, if the temperature drops below 20°C and the house is occupied, turn on the heating.

Initiating a Determinator

A Determinator can be initiated either by reception of a particular xPL message, or by the value of a global variable changing.
In addition to specifying the condition that initiates the determinator, you can specify supplemental conditions which must also be true, for example, you may want a different set of actions to occur based on whether the house is occupied or empty.

Determinator Actions

If a particular Determinator's conditions evaluate to true, any actions that have been specified will be executed.
Actions are executed in a user-specified order, and it is possible to insert pauses in the execution if required.

The following actions are currently supported:
  • Send an xPL Message
  • Set the value of a global variable (including changing the Mode/Period)
  • Run a sub-routine
  • Run a program
  • Execute another rule
  • Pause execution for a specified number of seconds
